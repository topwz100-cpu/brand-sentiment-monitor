## 数据流

### 搜索流程

```
用户输入关键词
    │
    ▼
前端发送搜索请求
    │
    ▼
后端接收请求
    │
    ▼
调用搜索服务
    │
    ├── Tavily API搜索
    │
    ├── 微博搜索
    │
    ▼
合并搜索结果
    │
    ▼
情感分析
    │
    ▼
返回结果
    │
    ▼
前端展示
```

### 定时任务流程

```
定时触发（每天6点）
    │
    ▼
获取所有活跃品牌
    │
    ▼
遍历品牌搜索新闻
    │
    ▼
情感分析
    │
    ▼
计算热度分数
    │
    ▼
生成Top20
    │
    ▼
保存到数据库
    │
    ▼
发送钉钉通知
```

## API设计

### RESTful API

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | /brands/ | 获取品牌列表 |
| POST | /brands/ | 创建品牌 |
| GET | /brands/{id} | 获取品牌详情 |
| PUT | /brands/{id} | 更新品牌 |
| DELETE | /brands/{id} | 删除品牌 |
| GET | /news/search | 搜索新闻 |
| GET | /news/brand/{id} | 获取品牌新闻 |
| GET | /news/latest | 获取最新新闻 |
| GET | /news/negative | 获取负面新闻 |
| GET | /dashboard/top20 | 获取Top20 |
| GET | /dashboard/stats | 获取统计数据 |
| GET | /dashboard/trends | 获取趋势数据 |
| GET | /dashboard/brand-distribution | 获取品牌分布 |

## 安全设计

### 认证授权

- 目前版本无认证（内部使用）
- 后续版本添加JWT认证

### 数据保护

- 环境变量保护敏感信息
- 数据库连接使用SSL
- API请求使用HTTPS

### 输入验证

- 使用Pydantic进行数据验证
- SQL注入防护（ORM）
- XSS防护（前端框架）
